# API 版本管理与兼容性指南

## 版本策略

QueryCraft 采用语义化版本管理策略，API 版本通过 URL 路径进行管理：
- `/api/v1/` - 当前稳定版
- `/api/v2/` - 预备新版本（包含新功能和改进）

## 向下兼容性承诺

我们承诺在同一个主版本内保持向后兼容性，即 v1.x.x 内的更改不会导致破坏性变更。

### 兼容性级别
- **完全兼容**: 添加新功能或扩展，不影响现有功能
- **部分兼容**: 修改非公开接口，对外部使用者无影响
- **向后兼容**: 功能仍可运行但在逐渐弃用状态
- **不兼容**: 重大更改导致原有功能无法使用

## 版本生命周期

版本遵循生命周期：

1. **发布** - 最新功能版本
2. **推荐使用** - 主要推广的版本
3. **维持** - 不添加新功能，只提供 bug 修复
4. **弃用** - 不再维护，建议用户迁移到新版本
5. **停用** - 关闭服务

## 代码实现

### 版本路由管理

版本管理通过 `version.py` 模块统一处理：

- 各个 API 版本通过独立的路由器（`APIRouter`）管理
- 版本协商中间件处理版本协商逻辑
- 弃用版本会自动添加警告头通知使用者

### API 调用指导

#### 现有用户

现有用户无需修改代码即可继续使用当前 API（如 `/api/v1/generate`），但建议逐步迁移到新版本获取更好的功能和性能。

#### 新用户

新用户推荐使用最新的稳定版本，API 文档会明确标注各版本的区别和使用方式。

## 优势

- **平滑升级**: 旧版本继续可用，给予用户充分时间迁移
- **风险可控**: 新版本独立测试，不影响已有应用
- **渐进改进**: 可以针对不同版本进行差异化优化
- **透明管理**: 用户随时了解 API 生命周期状态
